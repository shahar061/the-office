# Code of Conduct

We're building this for fun and for craft. The goal is a project where smart, kind people enjoy contributing.

## In short

- **Be kind.** Assume good faith. Default to generosity in interpretation.
- **Be specific.** "This doesn't work" is unhelpful; "Build fails on macOS arm64 with this stack trace" is gold.
- **Disagree on the work, not the person.** "I don't think this approach scales because X" — yes. Personal attacks — no.
- **Welcome newcomers.** Everyone is new to something. Today's confused contributor is tomorrow's maintainer.

## What's not OK

Harassment, discrimination, sexualized language or imagery, doxxing, sustained disruption of discussions, deliberate misgendering, posting others' private information without consent, trolling, or any conduct a reasonable person would find unprofessional in a working environment.

## What happens if it does

Maintainers may remove comments, close issues/PRs, or block accounts at their discretion. We aim to be calibrated, not punitive. If a first warning is enough, that's where we'll stop. If the behavior continues or is severe, the response will escalate accordingly.

## Reporting

If you experience or witness a violation, contact [shahar061](https://github.com/shahar061) directly via GitHub or open a [private security advisory](https://github.com/shahar061/the-office/security/advisories/new). All reports stay confidential. We'll acknowledge receipt within a few days and follow up with how we plan to respond.

## Scope

This applies to all project spaces — GitHub issues + PRs + discussions, the Discord server (when we have one), and any other forum where someone is representing the project.
